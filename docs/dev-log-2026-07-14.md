# Xuenwu 开发日志 2026-07-14

## 今日目标

在 TJ-Sylva 原有知识地图学习项目基础上，继续推进 Xuenwu 学生端能力，把“AI 学习助手、智能练习、错题本”拆清楚，并让智能练习从页面示例进入可开发、可验收的业务闭环。

## 需求确认

今日围绕智能练习模块重新梳理了 V1.0 范围，确定第一阶段不做教师端和管理端，重点先做学生端。

已确定的核心规则：

- 智能练习是独立左侧入口，不放在 AI 学习助手里面。
- 错题本也是独立入口，用于整理做错、不会、部分正确的题目。
- AI 学习助手负责根据学习路线、知识节点和错题情况给出建议，可以调用练习和错题数据。
- 智能练习每次默认生成 5 道题。
- 默认难度结构为 2 道基础题、2 道中等题、1 道综合题。
- 题目应更接近考试题，不能只是概念问答。
- 练习结果支持正确、部分正确、错误、不会、跳过。
- 练习完成后要形成反馈，并更新知识节点掌握状态。

## 已完成开发

### 1. 数据库模型

新增了智能练习相关数据结构：

- 练习批次表 `practice_sessions`
- 练习题目表 `practice_items`
- 作答记录表 `practice_attempts`
- 节点掌握状态表 `node_mastery`

同时扩展错题本字段，使错题可以关联练习来源、题型、难度、结果、复习状态等信息。

### 2. 后端接口

新增智能练习核心接口：

- 创建练习批次
- 查询练习批次详情
- 提交单题作答结果
- 完成本组练习并生成反馈
- 查询学习地图下的节点掌握状态

旧的复习题接口仍保留，避免影响已有功能。

### 3. 前端页面

左侧已经形成三个独立入口：

- `AI 学习助手`
- `智能练习`
- `错题本`

其中 `智能练习` 和 `错题本` 页面会隐藏右侧知识地图区域，让页面更专注。

智能练习页面已接入新的后端练习接口：

- 点击生成练习时，会创建一个练习批次。
- 每道题可以填写答案或思路。
- 每道题可以标记正确、部分正确、错误、不会、跳过。
- 做错、部分正确、不会的题目会进入错题相关流程。
- 完成本组练习后，可以生成本组练习反馈。

### 4. README 更新

README 已补充 Xuenwu 当前能力说明，包括：

- AI 学习助手
- 智能练习
- 错题本
- 基于原 TJ-Sylva 项目的二次学习与改造说明
